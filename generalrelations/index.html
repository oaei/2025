GeneralRelation

General description

The GeneralRelation track evaluates ontology matching systems on their ability to detect general semantic relations beyond simple equivalence. This better reflects real-world ontology alignment scenarios, where ontologies may differ in structure, granularity, and classification perspective.

Five mutually disjoint relation types are included:

  • Equivalence (≡) – Classes represent the exact same concept and share all instances.
  • Superclass_of (≥) – Every instance of the target class is also an instance of the source class (generalization).
  • Subclass_of (≤) – Every instance of the source class is also an instance of the target class (specialization).
  • Overlap (≈) – Classes share some but not all instances; their sets intersect without one being a subset of the other.
  • Disjoint (⊘) – Classes have no common instances; their sets are mutually exclusive.

This track addresses the need for alignment approaches capable of capturing nuanced relationships, enabling richer semantic integration for tasks such as knowledge graph merging, master data integration, and semantic search.

Datasets

The benchmark dataset suite is publicly available via Zenodo:

  • Zenodo Record – GeneralRelation Benchmark

The datasets come from three major sources: Product Classification Standards (ETIM, eClass, UNSPSC, GPC), STROMA/TaSeR Test Cases, and Modified OAEI Testcases. They vary in size, structure, and semantic complexity, and all reference alignments are annotated with explicit relation types.

Data Features

  • Large and heterogeneous class hierarchies with rich property structures.
  • Multi-source mappings combining exact matches and broader/narrower matches.
  • Support for n:m correspondences and partially correct alignments.
  • Reference alignments annotated with explicit relation types (≡, ≥, ≤, ≃, ⊘).
